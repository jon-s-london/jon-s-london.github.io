Production Monitoring

23/06/2022

This was a Python/Flask/HTML/CSS/sqlite3 project. It was also the final project for the Harvard CS50 course I completed.

The goal of the project was to provide ongoing monitoring for automated machinery 24/7 when no staff where present. The data was checked for suitability, stored in a database and then checked routinely with 'alarms' being raised for any points that staff should look at when next available.

The project code worked well - unforunately the incoming data was from a suppliers software which created CSV's that python then read. This software was deemed too unreliable and the supplier is working on an API system to replace it.

A v2.0 of this project is likely once that API is supplied and available as alot of value was added via this tool.

Customer Report Fixer

21/08/2022

This program is a time saving tool developed to take customer reports and batch sort them. Reports arrive in excel format, but with empty rows at the top of the worksheet, empty rows in the middle of the data and the data is formatted with headings on the left hand side.

The excel sheet is designed to be read from this document - but over a month there are hundreds or ever thousands of pieces of valuable data available - being able to use put the information together and then use tools like Excel or Power BI adds value to the data and turns it into information.

A Python CLI tool was developed to take the data, delete the un-needed rows and load it into a Pandas Dataframe before sorting and amending the data to a 'master document' that builds over time.

This led the information flow within the business to improve, the data to show the return on investments that had been made and analysis on trials to be conducted within minutes rather than hours of painstackingly manipulating and sticking together excel documents.
